플레이어 설정 구성

플레이어 설정은 사용되는 플레이어와 경험에 대한 플레이어 동작을 제어합니다. 플레이어 설정을 구성하려면 경험을 편집한 다음설정> 플레이어왼쪽 탐색에서

선수 선택

인페이지 경험에 사용할 플레이어를 선택하려면 사용가능한 플레이어드롭다운 목록에서 플레이어를 선택합니다. 목록에는 Video Cloud 계정의 모든 플레이어가 표시됩니다.

플레이어를 선택하면 플레이어 미리보기에 선택한 플레이어의 미리보기가 표시됩니다.

참고: 인 페이지 경험을 사용하는 경우 플레이어 스타일은 동영상이 재생되는 동안에 만 반영됩니다. 플레이어 스타일링은 이전 플레이 또는 애프터 플레이 상태에서는 표시되지 않습니다.

플레이어 행동 설정

플레이어 동작을 사용자 지정하도록 플레이어 설정을 구성 할 수 있습니다. 다음 설정을 사용할 수 있습니다.

  • 비디오 재생이 완료되면 비디오 재생이 완료될 때 어떤 일이 발생하는지결정합니다.
    • 경험에 대한 "재생 후" 상태 표시 - 선택 시 경험치가 입력됩니다. 플레이 후비디오가 완료되면 상태를 나타냅니다.
    • 현재 비디오에 유지 - 이 옵션을 선택하면 동영상이 완료될 때 환경이재생상태로 유지됩니다. 플레이어에 대해 구성된 사용자 지정 최종 화면 (플레이어 모듈에서 구성됨)을 표시하려면이 옵션을 선택합니다. 재생상태 중에 표시된 구성 요소도 계속 표시됩니다. 이 옵션은 경험에서 AFTER PLAY상태를 효과적으로 제거하는 부작용이 있습니다.
  • iOS 장치에서 인라인 재생 활성화 - 이 옵션을 선택하면 브라이트코브 플레이어 버전 5.10.5 이상을 사용할 때 iOS 장치에서 인라인 재생을 활성화합니다.
  • 자동으로 다음 비디오로 이동 -체크하면 현재 동영상 재생이 끝나면 경험의 다음 동영상이 재생됩니다. 이 설정은 After PLAE상태에서 카운트다운 타이머를 표시할지 여부도 결정합니다. 활성화되면 타이머가 있습니다. 타이머가 0에 도달하면 경험이 다음 비디오의재생상태로 전환됩니다.

IP 제한 계정 작업

북미이외의 IP 제한 계정은base_url플레이어에서 Playback API를 수정해야 합니다. 인페이지 경험은 경험이 생성되고 경험에 대해 플레이어가 선택될 때base_url플레이어에서 복사합니다. 기존 경험에 대한 플레이어의base_url온이 변경되면 Gallery는 이 값이 변경되었음을 알지 못하기 때문에 이 값을 자동으로 업데이트하지 않습니다. 해결 방법으로 게시자는 해결 방법으로 다음 단계를 따라야합니다.

  1. 경험을 위해 플레이어 설정 페이지로 이동하십시오.
  2. 현재 선택된 플레이어가 아닌 다른 플레이어를 선택하십시오.
  3. 초기 플레이어를 다시 선택하십시오.
  4. 구하다그리고출판하다경험.

이렇게 하면 초기 플레이어의 경험과 일치하도록 이 (가) 업데이트됩니다. base_url

완료되면저장을클릭하여 변경 사항을 저장합니다.

비디오 설정 구성

비디오 설정은 일부 비디오 표시 옵션을 제어하고 비디오를 다운로드 할 수 있는지 여부를 제어합니다. 이러한 설정은 비디오가재생상태일 때만 적용됩니다. 비디오 설정을 구성하려면 경험을 편집한 다음설정> 동영상왼쪽 탐색에서

관련 링크

선택하면 비디오 관련 링크가 비디오 정보와 함께 나타납니다. 관련 링크는 미디어 모듈의비디오 속성의 일부로 구성됩니다 .

기술

선택하면 동영상 정보와 함께 간단한 설명 대신 동영상에 대한 자세한 설명이 표시됩니다. 짧은 설명과 긴 설명은 미디어 모듈의비디오 속성의 일부로 구성됩니다 .

시청 표시기

체크하면 감시 표시기()은 사용자 세션 동안 시청한 비디오에 나타납니다.

참고: 감시된 표시기는 스타일 메뉴의 OVERLY BANNER속성을 사용하여 스타일이 적용됩니다.

다운로드

시청자가 동영상을 다운로드 할 수 있도록 인 페이지 경험을 구성 할 수 있습니다. 다운로드 링크를 통해 뷰어는 최고 품질의 변환을 다운로드 할 수 있습니다. 다운로드 옵션을 선택합니다.

  • 비디오 다운로드 허용 안함
  • 각 비디오에 대한 다운로드 링크 제공
  • 사용자 정의 필드 값을 기반으로 각 비디오에 대한 다운로드 링크 제공-사용자 정의 필드를 정의한 다음 다운로드 할 수있는 모든 비디오에 대해 해당 필드를 true로 설정합니다. 각 비디오에 대한 다운로드 링크를 구성하는 전체 단계는 인페이지 환경을 위한 비디오 다운로드구성을 참조하십시오 .

완료되면저장을클릭하여 변경 사항을 저장합니다.

소셜 설정 구성

소셜 설정은 비디오가 재생 될 때 사용할 수있는 소셜 공유 옵션을 제어합니다. 경험이 진행 중일 때 소셜 공유 아이콘이 플레이어 위에 나타납니다. 놀이상태. 비디오가 소셜 미디어 사이트에 공유되면 뷰어를 다시 경험으로 안내하는 링크 또는 이미지가 나타납니다.

갤러리는 소셜 미디어 메타 데이터 프록시 서비스를 사용합니다. 이 서비스는 동영상이나 이벤트가 인 페이지 경험에서 공유 될 때마다 사용되며 공유되는 URL의 일부입니다. 인 페이지 경험이 삽입 된 페이지로 시청자를 리디렉션하지만 크롤러는 대신 동영상 / 이벤트에 대한 메타 데이터를 보게됩니다. 이러한 방식으로 소셜 미디어 크롤러는 공유중인 동영상 / 이벤트에 대한 정보를 얻고 최종 사용자는 원래 페이지에 계속 액세스 할 수있는 동안 소셜 미디어 사이트에서 최종 사용자에게 더 관련성 높은 정보를 제공 할 수 있습니다.

디스플레이

표시 설정은 소셜 아이콘이 표시되는 방식을 결정합니다. 공유옵션 ()를 선택하면 공유 아이콘이 플레이어의 오른쪽에서 슬라이드 안으로 들어옵니다.

소셜 아이콘 옵션()을 선택하면 소셜 아이콘이 플레이어 위에 나타납니다.

공유 옵션

표시 할 소셜 공유 옵션을 선택합니다. 일부 플랫폼은 사용자 지정 메시지를 입력하는 기능을 제공합니다. 사용자 지정 메시지를 입력하려면 소셜 플랫폼 이름을 클릭하십시오. 링크를 클릭하여 메시지에있는 비디오 URL, 이름 또는 간단한 설명 값을 사용하십시오.

사용자 지정 메시지 설정을 숨기려면 소셜 플랫폼 이름을 다시 클릭하십시오.

SEO 설정에 액세스

SEO 설정을 사용하면 인 페이지 경험에 대한 사이트 맵 및 검색 피드를 생성 할 수 있습니다. SEO 설정에 액세스하려면 경험을 편집한 다음설정> 검색 엔진 최적화왼쪽 탐색에서

  1. 페이지 URL을 입력합니다 . 인 페이지 경험이 게시 될 사이트의 URL입니다.

사이트 세부 정보 구성

사이트 세부 정보 설정을 구성하여 검색 엔진에서 포털을 인덱싱하는 방법을 제어 할 수 있습니다. 검색 엔진 최적화 설정에 액세스하려면 환경을 편집하고사이트 구성> 검색 엔진 최적화왼쪽 탐색에서

다음 설정을 구성 할 수 있습니다.

  • 검색 엔진 결과 및 사이트가 공유 될 때 사이트 설명 사용 -체크하면 <meta> 설명 태그에 Facebook 및 Twitter 메타 태그를 포함한 사이트 설명이 사용됩니다. 선택하지 않으면 추천 동영상에 대한 설명이 사용됩니다.
  • 사이트 이미지 -포털 사이트의 홈페이지를 페이스 북이나 트위터에서 공유 할 때 표시되는 이미지입니다.
  • 키워드 -모든 사이트 페이지에 추가 될 메타 키워드. “새, 동물, 물고기, 동물 세계”를 추가하면 HTML 페이지에 다음이 추가됩니다.
    <meta name="keywords" content="birds,animals,fish,animal world">
  • 메타 태그 -사이트에 대한 추가 메타 태그를 추가하는 데 사용됩니다. 이러한 태그는 모든 사이트 페이지에 추가됩니다. 예:
    <meta name="myCustomTag" content="myCustomValue">
  • 페이지 메타 데이터에 사용자 정의 비디오 필드 포함 -체크시 사용자 정의 필드를 메타 태그로 추가합니다. 예를 들어 값이 2018인 사용자 지정연도필드는 다음을 추가합니다.
    <meta name="year" content="2018">
  • 페이지 메타데이터에 정식 URL 포함 - 이 옵션을 선택하면 소스에rel=canonical메타 태그를 추가하여 SEO를 개선할 수 있습니다. 이 기능은 Google에 동일한rel=canonical링크로 모든 페이지를 단일 항목으로 처리하도록 지시하는 기능입니다. 예를 들어, mysite.com/비디오/마이패지.html은 Mysite.com/비디오/마이페이지와 동일합니다.
  • URL에서 비디오 제목 제거 -선택하면 동영상 이름이 동영상 상세 페이지에 대한 URL의 일부로 포함되지 않으며 동영상 ID 만 URL에 포함됩니다.

사이트 맵 및 검색 피드보기

사이트 맵 설정은 사이트 맵 및 검색 피드에 대한 URL을 표시합니다. 사이트 맵을 다운로드하여 검색 엔진에 제출할 수 있습니다. 검색 피드는 대부분의 검색 엔진 소프트웨어에서 갤러리 콘텐츠를 검색 결과에 추가하는 방법으로 사용할 수 있습니다. 검색 엔진 최적화 설정에 액세스하려면 환경을 편집하고사이트 구성> 검색 엔진 최적화왼쪽 탐색에서

사이트 맵에 비디오 태그를 포함하려면 사이트 맵에 비디오 태그포함옵션을 선택합니다. 처음 32 개의 태그 만 포함됩니다. 사이트 맵에 태그를 추가하면 방문자가 비디오 태그를 기반으로 비디오를 찾을 수 있도록 검색 엔진 웹 크롤러에 태그가 노출됩니다.

참고: 이러한 URL은 사이트가 게시될 때까지 사용할 수 없습니다.

사이트 맵은 24 시간 동안 캐시됩니다. 24 시간 후 캐시에서 제거되고 다시 액세스 할 때 다시 생성됩니다. 비디오가 포털 경험에 추가되면 사이트 맵이 다시 생성되거나 (24 시간 후) 사이트가 다시 게시 될 때까지 사이트 맵에 포함되지 않습니다.

사이트맵 제출을 빌드하는 방법에 대한 자세한 내용은 Google 설명서를 참조하십시오 .
