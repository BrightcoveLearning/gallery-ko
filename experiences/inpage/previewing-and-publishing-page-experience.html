인페이지 경험 미리보기 및 게시

{{ page.description }}

인 페이지 환경은 디자인 중에 언제든지 미리 볼 수 있으며 다양한 장치 유형에서 어떻게 표시되는지 확인할 수 있습니다. 게시 대화 상자는 경험에 대한 변경 사항을 게시하는 기능을 제공하고 웹 페이지에 추가 할 소스 코드도 생성합니다.

인 페이지 경험 미리보기

인 페이지 경험을 미리 보면 갤러리 인터페이스 내에서 보지 않고도 경험이 어떻게 표시되는지 확인할 수 있습니다.

  1. 인페이지 환경을 미리 보려면 경험을 편집한 다음미리 보기를 클릭합니다 .
  2. 새 브라우저 탭에서 데스크톱 미리보기가 열립니다.
    참고: 인 페이지 경험을 사용하는 경우 플레이어 스타일은 동영상이 재생되는 동안에 만 반영됩니다. 플레이어 스타일링은 이전 플레이 또는 애프터 플레이 상태에서는 표시되지 않습니다.
  3. 왼쪽 상단 모서리의 드롭 다운 목록을 사용하여 다양한 장치 유형 및 다양한 환경 상태에서 환경을 미리 볼 수 있습니다.

인 페이지 경험 게시

인 페이지 환경이 변경 될 때마다 변경 사항이 게시되어 사이트에 표시되어야합니다. 경험을 변경하고 게시하지 않으면 변경 사항이 프로덕션 사이트에 표시되지 않습니다.

참고: 게시 대화 상자의공유 가능한 URL및포함 코드는인페이지 환경이 처음으로 게시될 때까지 사용할 수 없습니다.
참고: 인페이지 경험에 대한 변경 사항은 게시된 페이지에 나타나는 데 최대 5분이 걸릴 수 있습니다.

인 페이지 경험을 게시하려면 다음 단계를 따르십시오.

  1. 딸깍 하는 소리게시 및 포함페이지 상단에 있습니다. 게시 대화 상자가 열립니다.
  2. 변경 사항게시를 클릭합니다 . NS 출판 현황 "로 변경됩니다. 게시 진행 중… ".
  3. 게시가 완료되면게시 상태를 확인합니다 . 상태는이 경험이 최신 상태여야합니다.  환경이 업데이트되는 데 몇 분 정도 걸릴 수 있습니다.

환경이 변경되면 해당 환경을 다시 게시해야합니다.

소스 코드 복사

사이트에 페이지 내 환경을 표시하려면포함코드가 필요합니다. 경험이 처음 게시될 때임베드 코드를 복사하여 사이트에삽입할 수 있습니다. 환경이 변경되고 게시되면 해당 변경 사항이 사이트에 표시됩니다.

참고: 페이지 내 환경을 변경하면 레이아웃 크기를 고정 크기에서 반응형으로 또는 그 반대로 변경하지않는 한기존 포함 코드를 바꿀 필요가 없습니다.

임베드 코드에는 두 가지 유형이 있습니다 .

  • 추천 -대부분의 경우 권장됩니다. 페이지 내 경험은 페이지의<div>태그 안에 표시됩니다. 권장 사항은 환경이 자체적으로 크기를 조정하고 페이지에서 반응하는 방식으로 작동하도록 허용하는 데 필요합니다.
  • JavaScript 없음 - 페이지 내 환경이 페이지의<iframe>태그 안에 표시됩니다. 일반적으로 이 작업은 임베드를 허용하지<script>않는 CMS 또는 다른 플랫폼에 경험을 임베드할 때 수행됩니다. 인 페이지 경험이 자바 스크립트 없음 임베드 안에 항상 적절하게 맞지 않을 위험이 높으며 경험에 스크롤바 또는 추가 공백이 있습니다. 이 옵션을 사용하면 일부 기능이 제대로 작동하지 않으므로 가능한 한권장 (JavaScript) 임베드 (JavaScript) 를 사용하여 임베드하는 것이 좋습니다. 예를 들어이 옵션을 사용하면 분석에서 올바른 URL을보고하지 않고 라이트 박스 재생이 전체 화면으로 전환되지 않고 iFrame의 크기로 제한됩니다. 재생 전 / 중 / 후 상태에서 경험의 높이가 변경되면 경험 하단에 추가 공백이 생기거나 스크롤 경험이 생깁니다.

포함 코드를 복사하려면 임베드 코드복사를 클릭합니다 .

모바일 장치에서 페이지 내 환경을 올바르게 확장하려면 주변 사이트에viewport메타 태그를 추가하는 것이 좋습니다. 예를 들면 다음과 같습니다.

<meta name = "viewport"content = "width = device-width, initial-scale = 1.0">

viewport태그에 대한 자세한 내용은이 문서를 참조하십시오 .

동적 비디오 재정의 사용

인 페이지 경험에서 비디오를 변경하려는 경우가있을 수 있습니다. 예를 들어, 경험이 임베드 된 사이트 영역에 따라 경험의 비디오를 변경해야 할 수 있습니다. 각각 다른 비디오 세트가있는 새로운 인 페이지 경험을 만드는 대신 하나의 경험을 만든 다음 임베드 코드를 수정하여 적절한 비디오를 경험에 동적으로 할당 할 수 있습니다. 동적 비디오 재정의 사용에 대한 자세한 내용은 페이지 내 환경에서 동적 비디오 재정의사용을 참조하십시오 .
