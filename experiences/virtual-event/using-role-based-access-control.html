사용하여 역할 기반 액세스 제어 사용

{{ page.description }}

사용자가 Virtual Event Experience에 로그인해야하는 경우 역할 기반 액세스 제어를 사용하여 역할에 따라 사용자에게 표시되는 비디오를 필터링 할 수 있습니다. 역할 기반 액세스 제어는 사용자 관리 시스템 (예: 등록 시스템, SSO) 에서 반환되는 사용자 객체에 대해 지정된 SAML 속성을 사용합니다. 규칙은 사용자 역할 및 Video Cloud 태그를 기반으로 정의됩니다. Brightcove는 모든 비디오에 이벤트를 식별 할 수있는 하나의 태그와 발생해야하는 역할 기반 필터링을위한 추가 태그가있을 것을 권장합니다.

역할 기반 액세스 제어를 구성하려면 먼저 콘텐츠를 제한하는 방법을 결정하고 적절한 역할 기반 액세스 구성 파일을 만듭니다. 다음 섹션에서는 몇 가지 샘플 구성이 제공됩니다. 다음으로 역할 기반 구성 파일을 경험과 연결합니다. 마지막으로 동영상에 적절하게 태그를 지정합니다.

참고: 비디오 컬렉션을 만들 때재생 목록은역할 기반 액세스 제어 필터링을 지원하지 않으므로 재생 목록을 컬렉션 기준에 사용할 수 없습니다. 대신사용자 지정기준을 사용하십시오.

역할 기반 액세스 구성 파일 만들기

역할 기반 액세스 제어는 별도의 역할 기반 액세스 구성 파일을 사용하여 구성됩니다. 파일 형식은 다음과 같습니다.


{
     "advancedRoleFilter": "true",
     "defaultIncludeTags": ["virtual_event"],
     "defaultIgnoreTags": ["premium"],
     "roleAttribute": "role",
     "valueDelimiter": ",",
     "ignoreByRole": {},
     "includeByRole": {},
     "allowByRole": {}
}

파일의 속성 값은 액세스가 경험에 적용되는 방식을 제어합니다.

  • AdvanceDroleFilter - true로 설정하면 역할 기반 액세스 제어 가능
  • defaultIncludeTags -가상 이벤트 경험에 표시 되려면 모든 동영상에 있어야하는 태그입니다. 이를 통해 가상 이벤트 경험의 계정에있는 비디오의 하위 집합 만 표시 할 수 있습니다.
  • defaultIgnoreTags -이 태그가있는 동영상은 역할에 속하지 않는 사용자에게 표시되지 않습니다.
  • roleAttribute -사용자 관리 시스템에서 돌아 오는 사용자 레코드의 SAML 속성. 모든 SAML 속성을 사용할 수 있습니다.
  • valueDelimiter -여러 역할을 구분하는 데 사용되는 구분 기호 (여러 역할이 반환 될 수있는 경우)
  • ignoreByRole - 사용자의 역할에 따라 추가 태그를defaultIgnoreTags무시하여 를 보강합니다. 이것은 사용자가 자신의 역할에 따라 볼 수있는 더 많은 비디오를 제거하는 데 사용됩니다.
  • includeByRole -역할이 할당 된 사용자는 지정된 태그가있는 동영상 만 볼 수 있습니다.
  • AllowByRole - 사용자의 역할에 따라 태그를 추가적으로defaultIncludeTags추가하여 를 보강합니다. 이것은 역할에 따라 사용자에게 더 많은 비디오를 표시하는 데 사용됩니다. 지정된 역할을 가진 사용자는 태그가 지정된 동영상과 여기에 나열된 태그를 볼 수 있습니다. defaultIncludeTags

단일 프리미엄 역할

이 시나리오에서 일반 사용자는 대부분의 콘텐츠를 볼 수 있지만 "premium"태그로 태그가 지정된 콘텐츠의 하위 집합은 "premium_user"역할을 가진 사용자 만 사용할 수 있습니다. 프리미엄 사용자는 일반 사용자가 볼 수있는 모든 콘텐츠를 계속 볼 수 있습니다. "virtual_event"태그를 사용하면 이벤트에 대해 특별히 태그가 지정된 계정의 콘텐츠 만 표시하도록 사이트가 제한됩니다.


{
     "advancedRoleFilter": "true",
     "defaultIncludeTags": ["virtual_event"],
     "defaultIgnoreTags": ["premium"],
     "roleAttribute": "role",
     "valueDelimiter": ",",
     "ignoreByRole": {},
     "includeByRole": {},
     "allowByRole": {
          "premium_user": ["premium"]
     }
}

계층 형 콘텐츠 액세스

첫 번째 시나리오와 비슷하지만 각 계층이 더 많은 콘텐츠에 액세스하고 하위 계층의 모든 콘텐츠를 볼 수있는 여러 계층이 있습니다. 사용자 역할은 "tier_1", "tier_2"및 "tier_3"이지만 이러한 역할 중 하나가없는 사용자에게는 계층화 된 콘텐츠가 표시되지 않습니다. 태그는 각각 "tier1", "tier2"및 "tier3"입니다.


{
     "advancedRoleFilter": "true",
     "defaultIncludeTags": ["virtual_event"],
     "defaultIgnoreTags": ["tier1", "tier2", "tier3"],
     "roleAttribute": "role",
     "valueDelimiter": ",",
     "ignoreByRole": {},
     "includeByRole": {},
     "allowByRole": {
          "tier_1": ["tier1"],
          "tier_2": ["tier1", "tier2"],
          "tier_3": ["tier1", "tier2", "tier3"]
     } 
}

콘텐츠 액세스 분할

이 예에서 두 사용자 역할은 해당 역할에 대해 특별히 태그가 지정된 비디오 만 볼 수 있습니다. 역할이없는 사용자는 다음 태그 중 하나의 콘텐츠를 볼 수 없습니다.


{
     "advancedRoleFilter": "true",
     "defaultIncludeTags": ["virtual_event"],
     "defaultIgnoreTags": ["tag1", "tag2"],
     "roleAttribute": "role",
     "valueDelimiter": ",",
     "ignoreByRole": {},
     "includeByRole": {
          "role1": ["tag1"],
          "role2": ["tag2"]
     },
     "allowByRole": {}
}

언어 별 세분화

분할 기반 접근 방식을 사용하면 사용자의 언어가 포함 된 SAML 속성을 기반으로 콘텐츠를 제공 할 수 있습니다.


{
     "advancedRoleFilter": "true",
     "defaultIncludeTags": ["virtual_event"],
     "defaultIgnoreTags": [],
     "roleAttribute": "language",
     "valueDelimiter": ",",
     "ignoreByRole": {},
     "includeByRole": {
          "en": ["english"],
          "es": ["spanish"],
          "jp": ["japanese"]
     },
     "allowByRole": {}
}

병합 된 규칙 예

이 인위적인 예는 다양한 옵션을 결합하는 방법을 보여줍니다. 일반 사용자는 "premium"태그가 지정되지 않은 "virtual_event"태그가 지정된 모든 콘텐츠를 볼 수 있습니다. "premium_user"역할을 가진 사용자는 모든 일반 콘텐츠와 프리미엄 콘텐츠를 볼 수 있지만 "프로모션"태그가 지정된 콘텐츠는 볼 수 없습니다. "trial_user"역할을 가진 사용자는 일반 콘텐츠 및 프리미엄 콘텐츠를 볼 수 있지만 "프로모션"태그가 지정된 콘텐츠도 볼 수 있습니다. "expired_user"역할을 가진 사용자는 프로모션으로 태그가 지정된 동영상 만 볼 수 있습니다.


{
     "advancedRoleFilter": "true",
     "defaultIncludeTags": ["virtual_event"],
     "defaultIgnoreTags": ["premium"],
     "roleAttribute": "role",
     "valueDelimiter": ",",
     "ignoreByRole": {
          "premium_user": ["promotional"]
     },
     "includeByRole": { "expired_user": ["promotional"]
     },
     "allowByRole": {
          "trial_user": ["promotional"],
          "premium_user": ["premium"]
     }
}

역할 기반 액세스 프로필 할당

콘텐츠에 대한 액세스를 제어하기 위해 역할 기반 액세스 프로필을 생성했으면이를 경험과 연결합니다.

  1. 액세스 제어 프로필을 클립 보드에 복사합니다.
  2. 갤러리 모듈을 엽니 다.
  3. 경험을 편집하십시오.
  4. 딸깍 하는 소리외모와 행동> 관습왼쪽 탐색에서
  5. 액세스 프로파일 구성을사용자 정의 템플릿 JSON필드에 붙여넣습니다.
    참고: 이 필드가 표시되지 않으면 역할 기반 액세스 제어에 대해 계정이 활성화되지 않은 것입니다.
  6. 저장을 클릭합니다 .

당신이경험을 다시 게시변경 사항이 나타납니다.

SAML 속성 노출

경험이 SSO / 등록 시스템 공급자의 역할 속성에 액세스하려면 SSO 설정이 올바르게 구성되었는지 확인하십시오.

  1. 갤러리 모듈을 엽니 다.
  2. 갤러리 홈 페이지에서설정링크를 클릭합니다.
  3. 왼쪽 탐색에서액세스 제어 프로파일을 클릭합니다 . 생성 된 액세스 제어 프로필 목록이 표시됩니다.
  4. Virtual Event Experience에서 사용중인 프로필의 편집 아이콘을 클릭합니다.
  5. 확인모든 속성 노출페이지 하단의 옵션이 선택되어 있습니다.
  6. 저장을 클릭합니다 .
