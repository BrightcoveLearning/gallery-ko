사용하여 시뮬레이션된 라이브 비디오 사용

{{ page.description }}

시뮬리브는 짧습니다 시뮬레이션 라이브 및 은 VOD 콘텐츠를 마치 라이브인 것처럼 최종 사용자에게 표시할 수 있는 방법입니다. 동시 콘텐츠는 콘텐츠를 사전 녹화하고 라이브 이전에 광범위한 테스트를 할 수 있으므로 이벤트 주최자에게 매력적입니다. 인코더 문제 또는 기타 물류로 인해 발생하는 기존 라이브 이벤트의 잠재적 인 실패로 인해 발생할 수있는 위험을 줄입니다. 동시 동영상은 동영상과 함께 카운트 다운 타이머를 표시하여 언제 라이브가 될지 알려줍니다. 아래 예는 추천 컬렉션의 시뮬레이션 동영상입니다.

기본 및 보조 컬렉션에있는 동영상의 경우 동영상 카드에 카운트 다운 타이머가 표시됩니다.

가상 이벤트 경험에서 시뮬레이션 콘텐츠를 사용하려면 다음을 수행해야합니다.

  • 동시 플레이어 플러그인 구성
  • 플레이어를 경험과 연결
  • 시뮬레이션 사용자 정의 필드 만들기
  • 시뮬레이션 비디오 속성 구성
참고: 카운트다운 타이머의 일부로 표시되는 텍스트는 구성 파일의labels객체에 있으며 쉽게 사용자 정의하거나 현지화할 수 있습니다.

동시 플레이어 플러그인 구성

가상 이벤트 경험에서 사용되는 플레이어에 대해 Simulive 플레이어 플러그인을 구성해야합니다. 플러그인을 구성하는 데 필요한 정보를 얻으려면 계정 관리자에게 문의하세요.

  1. 딸깍 하는 소리선수탐색 헤더에서 플레이어 모듈을 엽니다.
  2. Brightcove는 가상 이벤트 경험을위한 전용 플레이어를 만들 것을 권장합니다. 전용 플레이어를 생성하지 않은 경우 New Player를 클릭하고 플레이어의 이름을 지정한 다음저장을 클릭합니다 .
  3. 구성하려는 플레이어의 링크를 클릭하십시오.
  4. 딸깍 하는 소리플러그인왼쪽 탐색 메뉴에서
  5. 플러그인추가 > 사용자 지정 플러그인을 클릭합니다 .
  6. 플러그인 이름에 를입력합니다simulivePlugin .
  7. JavaScript URL및 CSS URL의 경우 계정 관리자가 제공한 값을 사용합니다.
  8. 저장을클릭하여 변경 사항을 저장합니다.
참고: 플레이어에 대한 변경 사항은 게시된 플레이어에게 나타나는 데 최대 5분이 소요될 수 있습니다.

커스텀 플레이어 구성

사용자 지정 플레이어를 사용하도록 가상 이벤트 경험을 구성하려면 다음 단계를 따르십시오.

  1. 갤러리모듈을 엽니다.
  2. 경험을 편집하십시오.
  3. 딸깍 하는 소리비디오 및 재생> 플레이어 및 리드 양식왼쪽 탐색에서
  4. 사이트별 플레이어사용옵션을 선택하고 Simulive 플러그인으로 구성된 플레이어를 선택합니다.
  5. 저장을 클릭합니다 .

시뮬레이션 사용자 정의 필드 만들기

사용자 정의 필드는 비디오를 사용할 수있는 날짜와 시간을 저장하는 데 사용됩니다. 시뮬레이션 사용자 정의 필드를 만들려면 다음 단계를 따르십시오.

  1. 탐색 헤더에서 ADMIN 을클릭한 다음비디오 필드 를 클릭합니다 . ADMIN 메뉴에 액세스하려면 계정 관리자 여야합니다.
  2. 사용자 지정 필드추가를 클릭합니다 .
  3. 다음 값을 사용하여 새 사용자 정의 필드를 만듭니다.
    • 이름 표시하기 -동시 시작
    • 내부 이름 -simulive_start
    • 유형 -텍스트
    • 설명 - 예상 형식은 YYY-MM-DTHH:MM:SS-HH:MM입니다.
  4. 필드저장을 클릭합니다 .
사용자 지정 필드 생성에 대한 자세한 내용은 사용자 지정 메타데이터 필드만들기를 참조하십시오 .

시뮬레이션 비디오 속성 구성

사용자 지정 필드 Simulive Start는 비디오를 사용할 수 있는 날짜와 시간을 보유합니다. 동시 시작 시간에 도달 할 때까지 비디오를 볼 수 없습니다. Simulive 동영상도 simulive 태그를 지정해야합니다. 비디오를 시뮬레이션 비디오로 구성하려면 다음 단계를 따르십시오.

  1. 열기미디어기준 치수.
  2. 비디오 타이틀을 클릭하여 속성을 엽니 다.
  3. 비디오 속성섹션을 찾아편집을 클릭합니다 .
  4. simulive 로 동영상에 태그를지정합니다.
  5. 저장을 클릭합니다 .
  6. 사용자 정의 필드섹션을 찾아편집을 클릭합니다 .
  7. Simulive 시작필드에 날짜 및 시간 값을 입력합니다. 필드의 올바른 형식은YYYY-MM-DDTHH:MM:SS-HH:MM어디YYYY-MM-DD적절한 경우 선행 0이 있는 연도, 월 및 날짜입니다. HH:MM:SS는 적절한 경우 선행 0이 있는 24시간 시, 분 및 초입니다. -HH:MM시간대 오프셋입니다(예: UTC-4의 경우 -04:00, UTC+8.5의 경우 +08:30 등).
    참고: Simulive 는 지정된 사용자의 로컬 컴퓨터 시간을 사용하여 로컬 컴퓨터 시간과 구성된 Simulive 시작 시간을 비교하여 이벤트가시작되어야 하는 시기를 결정합니다. 이러한 이유로 일부 사용자에게는 예상보다 일찍 이벤트가 시작되는 경우가있을 수 있습니다. 이는 일반적으로 이러한 사용자의' 컴퓨터 시간이 정확하지 않기 때문입니다.
  8. 저장을 클릭합니다 .

정책 키 값 업데이트

카운트 다운 타이머가 시뮬레이션 비디오에 표시 되려면 구성 파일의 정책 키 값을 검색 가능한 정책 키로 업데이트해야합니다. 이렇게하지 않으면 동영상에 카운트 다운이 표시되지 않습니다. 검색 가능 정책 키를 가져오는 단계는코드 샘플을 참조하십시오. 정책 키를 생성합니다 . 새 정책 키를 만든 후 구성 파일의policyKey값을 업데이트하고 환경을 다시 게시합니다.
