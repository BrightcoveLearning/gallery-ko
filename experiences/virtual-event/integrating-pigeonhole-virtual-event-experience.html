통합 비둘기 홀과 가상 이벤트 경험 통합

{{ page.description }}

Pigeonhole을 사용하여 가상 이벤트 경험에서 하나 이상의 비디오에 이벤트 기반 상호작용(라이브 Q&A, 설문조사, 설문조사 등)을 쉽게 추가할 수 있습니다. 일반적으로 라이브 이벤트를 위해 새 Pigeonhole이 생성 된 다음 Pigeonhole 기능을 추가하려는 각 비디오에 대해 세션을 생성 할 수 있습니다. 세션이 생성되면 iframe 내장 코드가 Video Cloud 사용자 정의 필드를 사용하여 각 비디오와 연결됩니다. 소스 코드를 사용하면 선택한 동영상의 동영상 플레이어 옆에 Pigeonhole 포드가 표시됩니다.

새로운 Pigeonhole 이벤트 및 세션 생성

라이브 이벤트를위한 새 Pigeonhole을 만든 다음 Pigeonhole 기능이 추가 될 각 비디오에 대한 세션을 추가해야합니다.

  1. Pigeonhole 계정에 로그인하십시오.
  2. + Pigeonhole 추가를클릭하고 사용할 계정을 클릭합니다.
  3. 이벤트 이름및암호를 입력합니다 .
  4. 이벤트의시작및종료날짜를 입력합니다. 이 날짜는 Pigeonhole이 활성화되는 기간을 제어합니다.
  5. 시간대및사용자 지정 브랜딩옵션 (해당되는 경우) 을 선택합니다.
  6. 계속을 클릭합니다 .
  7. 이벤트가 생성되었으므로 왼쪽 탐색에서 Agenda항목이 선택됩니다. 세션추가를 클릭하고 새로만들기를 클릭합니다 .
  8. 세션 유형Q&A를 선택합니다 (다른 세션 유형도 사용할 수 있음).
  9. 세션 이름을 입력합니다 .
  10. 시작및종료날짜 및 시간을 입력합니다.
  11. Q&A 설정을 적절히구성합니다.
  12. Q&A 추가를 클릭하고 세션이 생성되었는지 확인합니다.
  13. 필요에 따라 다른 세션을 만듭니다.

Pigeonhole Embed 코드 생성

Pigeonhole이 비디오 옆에 표시되도록하려면 Video Cloud 사용자 정의 필드를 사용하여 iframe 포함 코드가 비디오에 추가됩니다. 코드 형식은 다음과 같습니다.

<아이프레임 src="비둘기집 URL“>  <  /이프레임 >

비둘기 URL값을 얻으려면 다음 단계를 수행하십시오.

  1. 비둘기집에서이벤트 실행왼쪽 탐색에서
  2. 참석자초대섹션에서 링크를 클릭합니다.
  3. 모든 세션 목록이있는 새 브라우저 탭이 열리는 지 확인합니다.
  4. 적절한 세션에 대한 Enter 링크를 클릭합니다 (예: 이 Q&A 입력) .링크는 세션 유형에 따라 변경됩니다.
  5. 새 브라우저 탭에서 Pigeonhole이 열리는 지 확인합니다. Pigeonhole URL이 브라우저 주소 표시 줄에 표시됩니다.

비둘기 URL이 있는 iframe태그가 해당 동영상에 추가됩니다.

이벤트 설정 구성

Pigeonhole 이벤트에 대한 설정도 구성해야 SSO 정보를 사용하여 사용자 프로필을 만들 수 있습니다.

  1. 왼쪽 탐색에서추가 설정을 클릭합니다 .
  2. 참석자탭을 클릭합니다.
  3. 임베드를 통해 참석자 프로필 생성허용옵션을 선택합니다.
  4. 페이지의등록자 프로필섹션으로 스크롤합니다.
  5. 이 이벤트에 대한 등록자 프로필사용옵션을 선택합니다.

사용자 정의 필드 만들기

가상 이벤트 템플릿이 비디오에 대해 표시 할 Pigeonhole을 알기 위해 사용자 정의 필드가 사용됩니다. 사용자 정의 필드를 만들려면 다음 단계를 따르십시오.

  1. 비디오 클라우드 스튜디오에 로그인합니다.
  2. 탐색 헤더에서 ADMIN 을클릭한 다음비디오 필드 를 클릭합니다 . ADMIN 메뉴에 액세스하려면 계정 관리자 여야합니다.
  3. Chat Embed라는 새 사용자 지정 필드를 만듭니다 .
  4. 내부 이름이 chat_embed인지 확인합니다 .
  5. Type 을텍스트로 설정합니다 .
  6. 사용자 정의필드를 저장합니다.

Video Cloud에서 사용자 지정 필드를 만드는 방법에 대한 자세한 내용은 사용자 지정 메타데이터 필드만들기를 참조하십시오 .

비디오에 Pigeonhole 추가

가상 이벤트 템플릿의 비디오 옆에 Pigeonhole이 나타나도록하려면 다음 단계를 따르십시오.

  1. 열기미디어기준 치수.
  2. 비디오 속성을 열려면 비디오를 클릭하십시오.
  3. 사용자 정의 필드섹션을 찾아편집을 클릭합니다 .
  4. Pigeonhole 임베드 코드를 Chat 임베드사용자 지정 필드에 붙여넣습니다.
  5. 저장을 클릭합니다 .

사용자 SSO 정보를 Pigeonhole에 전달

가상 이벤트 경험이 SSO를 사용하여 보호되는 경우 사용자 SSO 정보를 Pigeonhole에 전달하여 사용자를 Pigeonhole 내부에서 추적 할 수 있습니다. 다음 섹션에서는 SSO 정보를 Pigeonhole에 전달하는 데 필요한 설정 및 단계를 안내합니다.

SSO 속성 매핑

올바른 SSO 속성이 매핑되어 사용자 환경에 전달 된 다음 Pigeonhole에 전달 될 수 있도록해야합니다. 통합 Pigeonhole은 다음 값을 찾습니다.

  • fname -사용자 이름
  • nameID -사용자의 이메일 주소

아래 예는 Okta SSO 관리 페이지 내부에서 노출되는 이러한 속성을 보여줍니다.

액세스 제어 프로필에서 속성 노출

환경을 보호하는 데 사용되는 액세스 제어 프로파일에 모든 속성 표시옵션이 선택되어 있는지 확인합니다. SSO 정보를 사용할 수없는 경우 사용자는 Pigeonhole에서 익명으로 표시됩니다.

  1. 갤러리 모듈을 엽니 다.
  2. 딸깍 하는 소리설정헤더에.
  3. 왼쪽 탐색에서액세스 제어 프로파일을 클릭합니다 .
  4. 경험을 보호하기 위해 사용중인 프로필의 수정 아이콘을 클릭합니다.
  5. 모든 속성 표시설정이 선택되어 있는지 확인합니다.

Single Sign-On 구성에 대한 자세한 내용은 액세스 제어 프로파일 구성을 참조하십시오 . 모든 것이 올바르게 구성된 경우 뷰어가 경험에 로그인 할 때 사용자 정보가 Pigeonhole에 표시됩니다.
