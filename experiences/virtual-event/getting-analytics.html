소개

특정 계정에 대한 계정, 뷰어 및 비디오 분석 외에도 장치 및 위치별로 집계 된 가상 이벤트에 대한보고 이벤트를 볼 수 있습니다. 이 정보는 시청자 참여를 극대화하기 위해 예정된 가상 이벤트를 더 잘 구성하는 데 도움이 될 수 있습니다.

이 데이터는 Analytics API를 사용하여 다운로드 할 수 있으며 JSON, CSV 또는 XLXS 형식으로 반환 할 수 있습니다. 다음 섹션에서는 Analytics API에 대한 간략한 소개를 제공합니다. 사용 방법을 이미 알고있는 경우 가상 이벤트에 대한 분석 데이터 가져 오기에 대한 정보를 보려면 다음 섹션으로 건너 뛸 수 있습니다.

Analytics API

다른플랫폼 API와 마찬가지로 애널리틱스 API는 비디오 클라우드 플랫폼과 프로그래밍 방식으로 상호 작용할 수 있는 RESTful API입니다. 요청에 포함 된 다양한 URL 매개 변수를 기반으로 분석 데이터를 반환합니다. 기본적으로 데이터는 JSON 형식으로 반환되지만 CSV 또는 XLXS 데이터가 스프레드 시트 앱에서 열리도록 요청할 수 있습니다.

이 문서에서는 가상 이벤트에 대한 분석 데이터를 얻기 위해 알아야 할 사항만 다룹니다. 다른 용도에 대해서는 일반애널리틱스 API 설명서를 참조하십시오 .

신청 방법

RESTful API에 대한 요청은 브라우저가 웹 페이지를 검색하기 위해 URL에 요청하는 것과 동일한 방식으로 URL로 이루어집니다. 그러나 API 요청을 브라우저에 간단히 붙여 넣을 수는 없습니다. 요청은 인증을 받아야만 데이터를받을 권한이 있음을 보여주기 때문입니다. 그러나이를 수행하는 데 도움이되는 많은 도구가 있습니다. 다음은 간단한 목록입니다.

  • 잠 잘 수 없음 (인기있는 크로스 플랫폼 앱)
  • 우편 집배원 (다른 인기있는 크로스 플랫폼 앱)
  • 온라인 API 테스터 Brightcove Learning Services에서 생성
  • api-tester.sh (Brightcove Learning Services에서 만든 컬 기반 쉘 스크립트)

요청 인증

데이터를 보호하려면 Analytics API 요청에 액세스 토큰이 포함 된 Authorization 헤더가 있어야합니다.

Authorization: bearer your access token

액세스 토큰은 어떻게 얻습니까? 액세스 토큰얻기에서 모든 것을 읽을 수 있지만 짧은 대답은 약간의 준비를 한 후에 위에 나열된 도구 (및 기타 많은 도구) 가 사용할 수 있다는 것입니다.

  5. 저장을 클릭합니다 .
  6. 새 대화 상자에클라이언트 ID 클라이언트 암호가 표시됩니다 . 다시 찾을 수있는 안전한 장소에 둘 다 복사하여 저장하십시오.

액세스 토큰

액세스 토큰은 클라이언트 자격 증명을 사용하여 다른 Brightcove API에서 검색됩니다. 직접 수행 할 수 있지만 이전 섹션에 나열된 모든 도구는 클라이언트 자격 증명을 가져 와서 액세스 토큰을 가져 와서 Analytics API 요청과 함께 전송 된 요청 헤더에 넣습니다.

Analytics API 요청

다음은 Analytics API 요청의 형식입니다.

http://analytics.api.brightcove.com/v1/data?accounts={account1_id,account2_id}
&from={yyyy-mm-dd}
&to={yyyy-mm-dd}
&dimensions=viewer,video,country,city,region,device_type,protocol
&fields=video,video.name,viewer,country,region,city,device_type,time,video_view,video_percent_viewed,video_seconds_viewed
&sort=-video_view&limit=20&format=csv
요청 URL에 줄 바꿈이 없어야 합니다. 가독성을 위해 여기에 줄 바꿈이 추가되었습니다.

필드

  • video
  • country
  • region
  • city
  • device_type

집계 된 메트릭

  • video_view : 플레이어가 video_view를 보낸 총 횟수입니다 ( video_view는그렇지 않습니다.시청자가 비디오를 일시 중지했거나 다시 시작할 때 전송되거나 페이지를 새로 고치지 않고 비디오를 여러 번 시청할 때 전송)

계산 된 필드

  • 비디오_초_조회수 :
    • 순서에 관계없이 Collector API에서받은 최고 범위 값
    • 20초 이상의 범위를 설명하는 이벤트는 애널리틱스 시스템에 의해 삭제되며, 플레이어가 20초 이하의 범위를 전송하도록 올바르게 계측되지 않은 경우 부정확한 값을 초래할 수 있습니다. 개요를 참조하십시오. Data Collection API
  • video_%_Viewed:video_초_조회/비디오_지속 시간*100 (비디오 지속 시간이 데이터 수집기로 전송되지 않은 경우 이 값이 누락되거나 부정확합니다)
  • 비디오_엔진_xx : xx 비디오_백분율_조회수의 총 조회 수 (xx = 1, 25, 50, 75 또는 100)
