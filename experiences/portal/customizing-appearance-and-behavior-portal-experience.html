포털 경험의 모양 및 동작 사용자 지정

{{ page.description }}

모양 및 동작 설정을 구성하려면 경험을 편집하고외모와 행동왼쪽 탐색에서 다음 설정을 사용할 수 있습니다.

  • 템플릿 설정 -사용중인 템플릿에 대한 설정 구성
  • 머리글과 바닥 글 -경험에 대한 사용자 지정 머리글 / 바닥 글 구성
  • 커스텀 -사용자 정의 CSS 및 JavaScript 파일 사용

템플릿 설정 구성

템플릿 설정을 구성하려면외모와 행동> 템플릿 설정왼쪽 탐색에서 템플릿 설정은 사용중인 템플릿에 따라 다릅니다. 각 템플릿과 관련된 설정에 대한 정보를 보려면 링크를 클릭하십시오.

  • 목록
  • 연대기
  • 큰 천막
  • 라이브 이벤트 포털

모든 템플릿에 템플릿 설정이있는 것은 아닙니다.

사용자 정의 머리글 / 바닥 글 HTML 구성

갤러리는 사이트에 대한 사용자 지정 머리글 및 / 또는 바닥 글을 사용하는 옵션을 제공합니다. 이는 포털 경험을 사용자 정의하는 기능을 제공하므로 회사 사이트의 모양과 느낌에 더 가깝게 일치합니다. 사용자 지정 머리글 / 바닥 글은 계정 수준에서도 구성 할 수 있습니다. 계정 수준에서 사용자 지정 머리글/바닥글을 구성하는 방법에 대한 자세한 내용은 갤러리 설정구성을 참조하십시오 .

사용자 정의 머리글 또는 바닥 글을 사용하도록 포털 경험을 구성하려면 다음 단계를 따르십시오.

  1. 사이트에 사용할 머리글 / 바닥 글을 디자인하십시오. 머리글 / 바닥 글에는 유효한 HTML, JavaScript 또는 CSS 코드가 포함될 수 있습니다.
  2. 딸깍 하는 소리외모와 행동> 머리글과 바닥 글왼쪽 탐색에서
  3. 헤더 HTML을사용자 정의 헤더 HTML필드에 붙여넣습니다.
  4. 사용자 정의 헤더 아래에 기본 헤더포함옵션을 선택하여 사용자 정의 헤더 아래에 기본 템플릿 헤더를 표시합니다.
  5. 바닥글 HTML을사용자 정의 바닥글 HTML필드에 붙여넣습니다.
  6. 사용자 지정 바닥글 아래에 기본 바닥글포함옵션을 선택하여 사용자 지정 바닥글 아래에 기본 템플릿 바닥글을 표시합니다.
  7. 저장을 클릭합니다 .

사용자 지정 머리글 및 바닥 글 사용에 대한 참고 사항

  • 갤러리는 머리글 / 바닥 글에 제공된 HTML에 대한 유효성 검사를 수행하지 않습니다.
  • 머리글 및 바닥 글 HTML에는 명시적인 문자 제한이 없습니다.
  • 외부 CSS 및 JavaScript 파일이 지원됩니다.
  • 코드는 반응 형 디자인 원칙을 따라야하므로 장치간에 적절하게 확장됩니다.
  • 사이트가 Bootstrap을 사용할 때 CSS 충돌을 보았습니다 (갤러리도 Bootstrap을 사용합니다). 충돌을 확인하고 해결하십시오.

사용자 지정 설정 구성

사용자 지정 설정은 사이트의 모양과 느낌을 사용자 지정하는 기능을 제공합니다. 사용자 지정 설정을 구성하려면외모와 행동> 관습왼쪽 탐색에서

다음 설정을 사용할 수 있습니다.

  • 사용자 정의 CSS URL - 스타일을 사용자 정의하기 위해 사용자 정의.css 파일의 위치를 지정하는 데 사용됩니다. 클릭 CSS 보기사이트의 스타일을 지정하는 데 현재 사용 중인 CSS 파일을 보려면 링크를 클릭하세요.
  • 사용자 지정 JavaScript URL - 사용자 정의.js 파일의 위치를 지정하여 사이트의 스타일 및/또는 동작을 사용자 지정하는 데 사용됩니다.
  • 사용자 지정 Favicon URL -사이트에서 사용할 파비콘의 위치를 지정합니다. 파비콘 지원을 제공하는 브라우저는 일반적으로 페이지의 브라우저 탭과 북마크 목록의 페이지 이름 옆에 파비콘을 표시합니다.
  • 맞춤형 모바일 앱 아이콘 -사이트를 디바이스 홈 화면에 저장할 때 홈 화면 아이콘으로 사용할 이미지를 선택합니다. Brightcove는 고해상도 (1024 픽셀 x 1024 픽셀 권장)를 업로드 할 것을 권장합니다. 정사각형 이미지와 갤러리는 이미지 크기를 조정하고 장치에 가장 적합한 버전을 사용합니다.
  • 사용자 지정 응답 헤더 -포털에 대한 요청에 대한 응답으로 보낼 HTTP 헤더를 입력합니다. 예를 들어 CSP (콘텐츠 보안 정책) 또는 HSTS (HTTP-Strict-Transport-Security)를 추가하기 위해 헤더를 추가 할 수 있습니다.
