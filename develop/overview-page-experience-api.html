소개

인 페이지 경험과 관련된 두 가지 API가 있지만 매우 다른 용도로 사용됩니다.

인 페이지 경험 API (REST)

인 페이지 경험 API는 프로그래밍 방식으로 인 페이지 경험을 관리 할 수있는 REST API입니다. 주로 Video Cloud 플랫폼을 CMS 애플리케이션과 통합하는 데 사용됩니다.

또한 전체보기 API 참조사용 가능한 요청에 대한 자세한 내용은

인 페이지 경험 클라이언트 API

In-Page Experiences Client API는 Brightcove Player API가 플레이어의 동작을 제어하는 데 사용되는 방식과 유사하게 런타임시 In-Page Experience의 동작을 관리하는 데 도움이되는 JavaScript 라이브러리입니다. API는 하나 이상의 Brightcove 임베디드 환경을 실행하는 모든 페이지에서 사용할 수 있습니다. 개요를 참조하십시오. 자세한내용은 페이지 내 경험 클라이언트 API입니다.

이 개요의 나머지 부분에서는 백엔드에서 인 페이지 경험을 관리하기위한 REST API를 다룹니다.

인페이지 경험에 대한 일반적인 소개는개요를 참조하십시오. 인페이지 경험생성, 편집 및 게시

다음 섹션에서는 REST API 요청의 4 개 그룹에 대한 개요를 제공합니다.

  • 경험담
  • 템플릿
  • 테마
  • 글꼴

경험담

경험 요청은 세 가지 하위 범주로 나뉩니다.

경험 관리
이러한 요청을 통해 계정에 대한 모든 경험 목록을 얻을 수 있습니다. 새 환경을 만들고 특정 환경을 가져 오거나 업데이트하거나 삭제할 수도 있습니다. 마지막으로 경험을 복제하여 새로운 경험을위한 시작점을 만들 수 있습니다.
경험 게시 / 게시 취소
이들은 모두 계정 또는 특정 템플릿에 대한 모든 템플릿 목록을 가져 오도록 요청합니다.
라이브 경험에 대한 상태 관리
이 두 요청을 사용하면 라이브 경험의 현재 상태를 가져오거나 해당 상태를 (예: 에서 (으) preEvent 로live 변경할 수 있습니다.

템플릿

템플릿은 경험의 전체적인 모양과 느낌을 제어합니다. 사용자 지정 템플릿은 만들 수 없지만 몇 가지 표준 템플릿이 있으며 시간이 지남에 따라 더 추가 될 수 있습니다. API를 사용하면 계정에서 사용할 수있는 템플릿 목록을 가져 오거나 특정 템플릿을 검색 할 수 있습니다.

테마

테마를 사용하면 템플릿을 사용자 지정하여 원하는 모양과 느낌을 만들 수 있습니다. API는 테마를 완전히 관리하고 생성, 가져 오기, 업데이트 또는 삭제하는 방법을 제공합니다.

글꼴

자신 만의 글꼴을 사용할 수 있습니다. 이러한 메서드를 사용하면 글꼴을 추가, 업데이트, 가져 오기 또는 삭제할 수 있습니다. CSS를 통해로드 된 웹 글꼴입니다.
