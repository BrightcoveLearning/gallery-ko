포털 경험을 처음 만들면 형식의 임의 도메인이 할당됩니다<site-slug>.bcvp0rtal.com . 예를 들어, http://site-148393.bcvp0rtal.com . 사이트가 게시되면 뷰어는이 도메인의 사이트에 액세스 할 수 있습니다. 갤러리는 Brightcove 제공 도메인 또는 사용자 지정 도메인에서 비디오 사이트를 게시하는 기능을 제공합니다.

갤러리 포털 경험을 여러 URL에 할당하여 게시자가 사이트를 복제하지 않고도 다른 기능에 동일한 사이트를 사용할 수 있습니다. 예를 들어, 캠페인에서 기본 비디오 포털로 사용하려는 비디오 사이트가있는 경우 서로 다른 URL을 브랜딩하고 모두 동일한 사이트에 할당 할 수 있습니다. 사이트 복제가 필요하지 않으며 더 이상 필요하지 않은 URL을 쉽게 삭제할 수 있습니다.

참고: SSL 인증서로 작업할 때 어떠한 경우에도 개인 키를 공유하거나 브라이트코브 지원으로 보내지 마십시오.

Brightcove 제공 도메인 사용

Brightcove에서 제공하는 도메인을 사용하면 IT 부서와 협력하여 사용자 지정 CNAME을 생성하거나 레지스트라에 새 도메인 이름을 등록하지 않고도 사이트에 대한 도메인을 빠르고 쉽게 생성 할 수 있습니다. "즉시"사용할 수 있으며 버튼 클릭으로 구현할 수 있습니다. Brightcove에서 제공하는 세 가지 도메인이 있습니다.

  • brightcovegallery.com
  • gallery.video
  • bcvp0rtal.com (기본적으로 사이트에 할당 됨)

모두 SSL을 지원합니다. SSL 사이트에 액세스하려면 URL에 https를 추가하기 만하면됩니다. 예를 들어, https://marquee.brightcovegallery.com

사용자 지정 도메인 사용 (게시자 등록)

등록한 사용자 지정 도메인에서 비디오 사이트를 호스팅 할 수도 있습니다. 이를 통해 URL을 브랜딩하고 현재 웹 사이트 / URL 명명 구조와 일관성을 유지할 수 있습니다. 이를 위해서는 도메인 이름을 등록한 다음 호스팅 공급자를 통해 사용자 지정 CNAME 레코드를 구성해야합니다. 참조아래 섹션 CNAME 레코드를 구성하는 방법에 대한 정보는

사용자 지정 도메인 생성

사용자 지정 도메인을 생성하려면 다음 단계를 따르십시오.

  1. 갤러리 모듈을 엽니 다.
  2. 갤러리 홈 페이지에서설정링크를 클릭합니다.
  3. 왼쪽 탐색에서사용자 지정 도메인 및 SSL링크를 클릭합니다.
  4. 사용자 지정 도메인만들기를 클릭합니다 .
  5. 사용할 도메인 옵션을 선택하십시오.
    • brightcovegallery.com - 사이트 도메인의 형식이 됩니다<your domain>.brightcovegallery.com .
    • gallery.video - 사이트 도메인의 형식이<your domain>.gallery.video 있습니다.
    • 커스텀 -사용자 지정 도메인에서 사이트를 호스팅하려면 선택합니다. 이 옵션을 선택할 때 호스팅 공급자와 CNAME 레코드도구성해야합니다.
  6. 도메인 이름을 입력하십시오. 도메인이 유효하거나 유효하지 않음을 나타내는 메시지가 표시됩니다. 도메인 이름이 유효하지 않은 경우 새 도메인 이름을 입력하십시오.
  7. 브라이트코브 제공 도메인을 사용하여 사이트를 게시하는 경우 이 도메인에 SSL 적용 옵션이있습니다. 이 도메인에 SSL 적용 (권장)을 선택하여 뷰어가 해당 사이트로 이동하는 경우 비SSL 버전의 사이트에서 리디렉션을 활성화합니다.
  8. 사이트를 사용자 지정 도메인에 게시하는 경우 SSL을 사용하여 사이트를 보호 할 수있는 옵션이 있습니다. SSL을사용하여 사이트를 보호하려면 SSL 사용확인란을 선택합니다. SSL 사용에 대한 자세한 내용은 SSL을 사용한 포털 환경보안을 참조하십시오 .
  9. 확인하다리디렉션 활성화이 URL을 다른 URL로 리디렉션합니다. 일반적으로 이것은이 도메인이 한동안 사용 된 후에 만 확인됩니다. 이 옵션을 선택하면 다음을 입력하십시오. 리디렉션 URL이 기존 URL은 리디렉션되어야 합니다. 영구 리디렉션사용 (HTTP 상태 코드 301)을 선택하여 리디렉션이 영구적임을 나타냅니다. 검색 엔진 스파이더가 웹 페이지의 응답 헤더에서 301 상태 코드를 발견하면이 웹 페이지가 더 이상 존재하지 않음을 이해하고 색인 된 URL을 새 URL로 바꿉니다. 이 옵션은 이전 웹 페이지 트래픽과 페이지 순위를 유지합니다.
  10. 저장을 클릭합니다 . 구성된 도메인 목록이 표시됩니다.
    참고: 브라이트코브 제공 도메인을 사용하는 경우 SSL 상태는 항상제공됨으로 표시됩니다 . 이는 사이트 URL의 시작 부분에 https: //를 추가하면 브라우저와 갤러리 사이트 간의 데이터 전송이 안전하다는 것을 나타냅니다.
  11. 필요에 따라 추가 도메인을 만듭니다.

도메인을 수정하거나 삭제하려면 수정() 또는 삭제() 버튼.

사용자 지정 도메인이 생성되면 사이트 속성의 일부로 사이트에 할당 할 수 있습니다. 자세한 내용은 포털 경험에 사용자 지정 도메인할당을 참조하십시오 .

호스팅 제공 업체로 CNAME 레코드 구성

참고: CNAME 레코드는 사용자 지정 도메인이 사이트에 할당 될 때만 구성하면됩니다. 브라이트코브 제공 도메인을 사용하는 사이트에 대해서는 CNAME 레코드를 만들 필요가 없습니다.

CNAME (정식 이름) 레코드를 구성하는 단계는 호스팅 공급자에 따라 다릅니다. 사용할 각 사용자 지정 도메인에 대해 CNAME 레코드를 구성해야합니다. 사이트가 SSL을 사용하도록 구성된 경우 CNAME 레코드는 CloudFront URL을 가리켜 야합니다. 보다 SSL로 포털 경험 보호정보를 위해. CNAME ex: 로 구성하려면 사용자 지정 도메인에 하위 도메인을video 포함해야 합니다. brightcovelearning.com CNAME.

CNAME 값은 사용자 지정 도메인 및 SSL 설정의 일부로 표시됩니다.

이 예에서는 의video.brightcovelearning.com 사용자 지정 도메인에 대해 CNAME레코드를 추가하고 호스트를 추가합니다. video 그리고 그것을 가리킵니다d3mkhzp6k6x3hl.cloudfront.net .

Cloudfront URL을 직접 검색하면 "페이지를 찾을 수 없음"오류 (404)가 반환됩니다. curl명령을 사용하여 도메인 구성을 확인할 수 있습니다. 이 예에서는 다음을 수행합니다.

curl -H "Host: video.brightcovelearning.com" https://d3mkhzp6k6x3hl.cloudfront.net

curl 문은 사이트 홈 페이지의 내용을 반환해야 합니다.

이 단계에 대한 도움이 필요하면 네트워크 관리자 나 웹 호스팅 제공 업체에 문의해야 할 수 있습니다. DNS 변경 사항이 적용 되려면 최대 48 시간이 소요될 수 있습니다.

SSL로 사이트 보안

SSL을 사용하여 포털 환경을 보호하는 데는 두 가지 옵션이 있습니다.

  1. 할당된 사이트 슬러그를 사용하여 게시되거나 브라이트코브가 제공하는 도메인을 사용하는 사이트의 경우 https를 지원하기 위한 구성이 필요하지 않습니다. <site-slug>.bcvp0rtal.com URL의 시작 부분에 https: //를 추가하면 브라우저와 갤러리 사이트 간의 데이터 전송이 안전하게 보호됩니다. 이 도메인에서 SSL 적용 (권장)을 선택하여 뷰어가 해당 사이트로 이동하는 경우 비SSL 버전의 사이트에서 리디렉션을 활성화할 수도 있습니다.
  2. 사용자 지정 도메인을 사용하여 호스팅되는 사이트의 경우 도메인 이름을 입력할 때 SSL 사용옵션을 선택합니다. 그러면 인증서 정보를 입력하라는 메시지가 표시됩니다. 자세한 내용은 SSL을 사용하여 포털 환경보안을 참조하십시오 .
