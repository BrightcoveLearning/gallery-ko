SSL로 포털 환경 보호

{{ page.description }}

SSL (Secure Sockets Layer)은 웹 서버와 브라우저간에 암호화 된 링크를 설정하기위한 표준 보안 기술입니다. 이 링크는 웹 서버와 브라우저간에 전달되는 모든 데이터가 비공개 및 통합 상태로 유지되도록합니다. SSL은 업계 표준이며 수백만 웹 사이트에서 고객과의 온라인 거래를 보호하는 데 사용됩니다.

참고: 기본적으로 모든 Brightcove 제공 도메인은 SSL을 지원합니다. 비디오 사이트에서 기본 할당된 URL 또는 브라이트코브가 제공하는 도메인 (brightcovegallery.com 또는 gallery.video) 을 사용하는 경우 URL에 https를 붙여서 사이트에 액세스할 수도있습니다. 또한 이 도메인에 SSL 적용 (권장) 옵션도 있습니다. 뷰어가 해당사이트로 이동하는 경우 SSL 이외의 버전의 사이트에서 리디렉션을 활성화할 수 있습니다. 설정 또는 SSL 인증서가 필요하지 않습니다. 이 항목에서는 사용자 지정 도메인과 함께 자체 SSL 인증서를 사용하는 방법에 대해 설명합니다.

갤러리에서 SSL 사용에 대한 참고 사항

  • 게시자는 SSL 인증서를 담당합니다.
  • 자체 서명 된 인증서는 지원되지 않습니다.
  • 개인 키는 암호로 암호화 할 수 없습니다.
  • 개인 키는 다음으로 시작하고 끝나야 합니다. ‘ --- -- RSA 개인 키 시작 ----- ’그리고‘ --- -- RSA 개인 키 종료 ----- ’
  • 갤러리에서 지원하는 최대 SSL 인증서 키 길이는 2048 비트입니다.
  • Brightcove에서는 인증서가 PEM 형식이어야합니다.
  • Brightcove는 갤러리 사이트에서 사용되는 SSL 인증서를 다른 곳에서는 사용하지 말 것을 권장합니다.
  • 게시자는 사용자 지정 도메인 이름을 사용해야합니다.
참고: SSL 인증서로 작업할 때 어떠한 경우에도 개인 키를 공유하거나 브라이트코브 지원으로 보내지 마십시오.

SSL 용 포털 경험 구성

이 단계에서는 SSL 인증서를 구입했다고 가정합니다. 비디오 사이트도 사용자 지정 도메인을 사용하도록 구성되어야합니다. 사용자 지정 도메인을 사용하도록 사이트를 구성하는 방법에 대한 자세한 내용은 사용자 지정 도메인 및 SSL 구성을 참조하십시오 .

SSL 인증서 업로드

  1. 갤러리 모듈을 엽니 다.
  2. 설정링크를 클릭합니다.
  3. 왼쪽 탐색에서사용자 지정 도메인 및 SSL링크를 클릭합니다.
  4. SSL을 사용할 사용자 지정 도메인을 편집합니다.
  5. SSL 사용을 선택합니다 .
  6. 서버 인증서에 붙여 넣습니다 . 키의 시작 또는 끝에있는 빈 줄을 제거하십시오. Brightcove에서는 인증서가 PEM 형식이어야합니다.
  7. 서버 비공개 키에 붙여 넣습니다 . 키의 시작 또는 끝에있는 빈 줄을 제거하십시오. Brightcove에서는 인증서가 PEM 형식이어야합니다.
  8. CA 인증서체인에 붙여 넣습니다 . 인증서가 루트 CA에서 서명 된 경우 루트 인증서 CA 만 붙여 넣으면됩니다.  여기에는 자체 인증서가 포함되지 않아야합니다. 인증서 주문은 즉시 인증서입니다. > 기타 모든 중개자> 루트 (루트 인증서는 선택 사항).
    키의 시작 또는 끝에있는 빈 줄을 제거하십시오. Brightcove에서는 인증서가 PEM 형식이어야합니다.
  9. 저장을 클릭합니다 . 사이트 세부 정보가 표시됩니다.
  10. 저장을 클릭합니다 . 인증서가 확인되고 오류가있는 경우 표시됩니다.

SSL 상태가 표시됩니다. 구성 업데이트 SSL 정보가 업데이트되고 전파되는 동안 목록새로 고침을 클릭하여 SSL 상태를 업데이트합니다.

CNAME 레코드 구성

SSL이 구성되면 SSL상태가활성화됩니다 . CNAME 정보는 CloudFront URL로 업데이트됩니다. 도메인이 삭제되지 않는 한 CloudFront URL은 변경되지 않습니다.

CloudFront URL로 직접 이동하면 "페이지를 찾을 수 없음"오류 (404)가 반환됩니다. curl명령을 사용하여 도메인 구성을 확인할 수 있습니다. 이 예에서는 다음을 수행합니다.

curl -H "Host: video.brightcovelearning.com" https://d3mkhzp6k6x3hl.cloudfront.net

curl 문은 사이트 홈 페이지의 내용을 반환해야 합니다.

사용자 지정 도메인의 CNAME 레코드는 사용자 지정 도메인 및 SSL 페이지에 표시된 CloudFront URL로 리디렉션되어야합니다.

참고: 사용자 지정 머리글과 바닥글을 사용하는 경우 모든 URL도 안전한지 확인합니다.

SSL 인증서 업데이트 / 교체

SSL 인증서를 업데이트하거나 교체해야 하는 경우 (예: SSL 인증서가 만료되도록 설정될 수 있음) 다음 단계에 따라 기존 SSL 인증서를 교체합니다.

  1. 갤러리 모듈을 엽니 다.
  2. 설정링크를 클릭합니다.
  3. 왼쪽 탐색에서사용자 지정 도메인 및 SSL링크를 클릭합니다.
  4. SSL 인증서를 사용하여 사용자 정의 도메인을 찾은 다음 편집 아이콘().
  5. 새 인증서업로드를 클릭합니다 .
  6. 새서버 인증서, 서버 개인 키및 CA 인증서체인에 붙여 넣습니다 . 키의 시작 또는 끝에있는 빈 줄을 제거하십시오.
  7. 저장을 클릭합니다 .

업데이트 된 키에 오류가있는 경우 이전 값이 유지됩니다.

자주 묻는 질문

  1. 맞춤 도메인 및 SSL이있는 갤러리 사이트는 HTTPS를 통해서만 액세스 할 수 있나요? 예, 사실 이것이 사이트 액세스를 허용하는 유일한 방법입니다. HTTP 링크에서 사이트에 액세스하려고하면 자동으로 HTTPS 사이트로 리디렉션됩니다. 이는 CloudFront에서 HTTPS 리디렉션을 강제함으로써 수행됩니다.
