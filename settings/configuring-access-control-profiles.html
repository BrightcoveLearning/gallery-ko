액세스 제어 프로필 구성

{{ page.description }}

기본적으로 갤러리 포털 환경이 게시되면 사이트 URL이있는 모든 사용자가 사이트에 액세스 할 수 있습니다. 갤러리 포털 경험에 대한 액세스를 제한하기 위해 액세스 제어 프로필을 만들 수 있습니다. 액세스 제어 프로필은 갤러리 설정의 일부로 생성 된 다음 프로필이 사이트에 할당됩니다. 액세스 제어 프로필을 생성하여 다음을 기반으로 액세스를 제한 할 수 있습니다.

  • IP 주소
  • 싱글 사인온
  • 액세스 코드

갤러리 포털 환경은 단일 액세스 제어 프로필로만 구성 할 수 있습니다. 각 액세스 제어 프로필에는 하나의 액세스 제한 유형 만있을 수 있습니다. 예를 들어, IP 주소와 액세스 코드를 모두 사용하는 프로필을 만들 수 없습니다. 또한 Single Sign-On을 사용하는 경우 프로필 당 하나의 ID 공급자 만 구성 할 수 있습니다.

액세스 제어 프로필을 만들려면 다음 단계를 따르십시오.

  1. 갤러리 모듈을 엽니 다.
  2. 갤러리 홈 페이지에서설정링크를 클릭합니다.
  3. 왼쪽 탐색에서액세스 제어 프로파일을 클릭합니다 . 생성 된 액세스 제어 프로필 목록이 표시됩니다.
    참고: 생성하지 않은 계정에 액세스 제어 프로필이 표시 될 수 있습니다. Gallery가 액세스 제어 프로파일을 구현하기 전에 IP 제한, SSO 또는 사이트에 대한 액세스 코드를 구성한 경우 해당 기능이 릴리스될 때 해당 설정이 액세스 제어 프로파일로 저장됩니다.
  4. 프로필만들기를 클릭합니다 .
  5. 프로필에이름을 지정합니다 .
  6. (선택 과목)구성세션 타임 아웃사용자의 세션이 지속되는 시간을 지정합니다. 몇 가지 참고 사항:
    • 이 설정은 SSO 및 액세스 코드를 사용할 때 적용됩니다.
    • 이것은 "유휴 시간 제한"입니다. 갤러리 서버에 도달 할 때마다 (사이트에 페이지를로드하거나 사이트에 대한 API 호출을 통해) 타이머가 재설정됩니다. 이를 통해 시청자는 지정된 제한 시간보다 오래 계속해서 탐색 할 수 있지만 제한 시간보다 오래 유휴 상태가되면 즉시 로그 아웃됩니다. 비디오를 시청하는 동안 갤러리 서버는 액세스되지 않으므로 시간 제한 값은 가장 긴 비디오보다 길어야합니다.
    • SSO 공급자는 강제 재 인증을 받거나 세션 시간 제한이 우리의 시간 제한보다 작도록 설정되어야합니다. 그렇지 않은 경우 시간 초과 후 SSO 공급자로 리디렉션되고 사용자가 로그인했다는 응답을 즉시 보냅니다.
  7. 사이트를 보호하기위한 옵션을 선택합니다. 액세스 제어 프로파일당 하나의 옵션만 선택할수 있습니다.
    • IP 주소
    • 싱글 사인온
    • 액세스 코드
  8. 저장을 클릭합니다 .
  9. 필요에 따라 추가 액세스 제어 프로필을 만듭니다.

액세스 제어 프로필을 수정하거나 삭제하려면 수정() 또는 삭제() 액세스 제어 프로필과 연결된 버튼. 액세스 제어 프로필이 생성되면 사이트 속성의 일부로 사이트에 할당 할 수 있습니다. 자세한 내용은 포털 경험에 대한 액세스제어를 참조하십시오 .

IP 제한 구성

IP 제한을 사용하는 액세스 제어 프로필을 만들 때 미리 정의 된 범위를 벗어난 IP 주소를 가진 뷰어가 사이트에 액세스하려고하면 오류가 발생합니다.

IP 제한을 사용하도록 액세스 제어 프로필을 구성하려면 다음 단계를 수행하십시오.

  1. IP 주소로제한옵션을 선택합니다.
  2. 사이트에 액세스 할 수있는 IP 주소 범위를 입력하십시오. CIDR 블록 표기법도지원됩니다.
  3. 저장을 클릭합니다 .

싱글 사인온 구성

SSO를 사용하는 액세스 제어 프로필을 만들 때 사이트 방문자가 사이트에 액세스하기 전에 유효한 자격 증명으로 로그인하라는 메시지가 표시됩니다. Gallery는 보안 도메인간에 인증 및 권한 부여 데이터를 교환하기 위해 SAML 2.0 (Security Assertion Markup Language 2.0) 표준을 지원합니다.

Gallery는 다음을 포함한 대부분의 주요 ID 제공 업체를 지원합니다.

  • 핑
  • OneLogin
  • ADFS 2.0 / 3.0
  • Google G Suite
  • 표어
  • Okta
  • Salesforce.com

SAML 2.0 표준을 사용하는 다른 공급자도 작동합니다. 갤러리는여권 saml도서관을 사용합니다. 갤러리는이 라이브러리에서 작동하지 않는 ID 공급자를 지원하지 않습니다.

참고: 갤러리에는 싱글 사인온 인증을 위해 SAML 2.0이 필요합니다. SAML 1.1은 지원되지 않습니다.
참고: 싱글 바이트 문자만 지원합니다.
참고: 갤러리는 IDP(ID 공급자 시작) SAML 흐름만 지원합니다. 서비스 제공자가 시작한 흐름은 지원되지 않습니다.

SSO 제공 업체에는 Assertion Consumer Service URL (ACS URL)이 필요합니다. ACS URL은 액세스 제어 프로필 표의 SSO 섹션에 나열됩니다.

ACS URL의 기본값은https://auth.bcvp0rtal.com/login/callback입니다. 자체 URL을 제공하여 ACS URL을 재정의 할 수도 있습니다. 이 두 옵션을 모두 더 엄격한 SSO 공급자를 구성하는 데 사용할 수 있습니다. 기존 사이트의 경우http(s)://<site-root-url>/login/callback는 계속 작동하지만 Brightcove는 게시자가 사용할 것을 권장합니다https://auth.bcvp0rtal.com/login/callback . 이 끝점은 도메인에서 경로를 사용하는 모든 사이트의 올바른 사이트로 SAML 데이터를 전달합니다.

참고: IDP 시작 SSO 워크 플로를 사용하는 경우 SSO 소프트웨어에 지정된 ACS URL은 해당 액세스 제어 프로필로 구성된 갤러리 사이트를 가리켜 야합니다. 이 경우https://auth.bcvp0rtal.com/login/callback URL을 사용해서는 안 됩니다.

SSO를 사용하도록 액세스 제어 프로필을 구성하려면 다음 단계를 수행하십시오.

  1. SSO옵션을 선택합니다.
  2. SAML 2.0 메타데이터의 내보내기 파일이 있는 경우 파일선택을 클릭하여 메타데이터 파일을 찾아서 선택합니다. 이렇게 하면 SAML 2.0 엔드포인트및 X.509 인증서필드가 미리 채워집니다.
  3. SAML 2.0 엔드포인트 (HTTP) 에 대한 값을 입력합니다 . 이 값은 SAML 제공 업체에서 제공해야합니다.
  4. X.509 인증서 값을 입력합니다 . 이 값은 SAML 제공 업체에서 제공해야합니다.
  5. (선택 사항) SAML 발급자/엔티티 ID 재정의를 입력합니다 . 여기에 값을 입력하여 SAML 발급자를 선택적으로 재정의합니다.
  6. (선택 사항) ACS URL 재정의를 입력합니다 . 선택적으로 ACS URL을 재정의하려면 여기에 값을 입력합니다.
  7. (선택 사항) SAML 인증컨텍스트 재정의를 입력합니다 . 여기에 값을 입력하여 SAML AuthnContext를 선택적으로 재정의합니다.
  8. (선택 사항) 기본 AuthnContext를 비활성화하도록 선택합니다 . 기본적으로 SSO 프로필의 경우 갤러리는 PasswordProtectedTransport를 SAML 인증 컨텍스트로 보냅니다. 때때로 이것은 ID 공급자가 지원하지 않습니다. 이 옵션을 선택하면 갤러리에서 인증 컨텍스트 요청을 보내지 않습니다.
  9. (선택 사항) 외부 자바 스크립트에 전달하려는 SAML 속성을 입력합니다.
  10. (선택사항) 확인모든 속성 노출외부 JavaScript에서 사용할 모든 SAML 속성을 노출합니다.
  11. 저장을 클릭합니다 .

액세스 코드 구성

사이트에 액세스 코드가 적용된 경우 사이트 URL에 액세스 할 때 시청자에게 올바른 액세스 코드를 제공하라는 메시지가 표시됩니다.

잘못된 액세스 코드를 입력하면 뷰어에 오류가 발생합니다. 액세스 코드는 건설중인 사이트에 대한 액세스를 제한하는 좋은 방법입니다. 액세스 코드는 사이트 보안 또는 인증을 제공하지 않습니다.

액세스 코드를 사용하도록 액세스 제어 프로필을 구성하려면 다음 단계를 따르십시오.

  1. 액세스 코드필요옵션을 선택합니다.
  2. 들어가다액세스 코드사이트를 위해.
  3. 액세스 코드 레이블을 입력합니다 . 사용자가액세스 코드를 입력할 위치 위에 액세스 코드 레이블이나타납니다.
  4. (선택 사항) 액세스 제어 페이지의스타일을지정합니다.
  5. 저장을 클릭합니다 .

액세스 제어 페이지 스타일 지정

액세스 제어 페이지의 스타일을 전체 사이트의 모양과 더 가깝게 맞출 수 있습니다. 갤러리는 세 가지 스타일 옵션을 제공합니다.

  • 기본
  • 이미지 및 테두리
  • 커스텀

기본 스타일

기본 스타일 옵션을 사용하면 갤러리 로고, 액세스 코드 레이블 및 액세스 코드 필드가 항상 표시됩니다.

이미지 및 테두리 스타일링

이미지 및 테두리 옵션은 액세스 제어 페이지에 대한 몇 가지 기본 사용자 지정을 허용합니다. 예를 아래에서 볼 수 있습니다.

이미지 및 테두리 스타일을 사용자 지정하려면 다음 단계를 따르십시오.

  1. 이미지 및 테두리를 선택합니다 .
  2. 액세스 제어 페이지에 표시 할 이미지에 대한 링크를 끌어서 놓거나 찾아 보거나 추가합니다.
  3. 클릭스루 URL을 입력합니다 . 클릭스루 URL은 이미지를 클릭 가능한 링크로 만듭니다.
  4. 색상 선택기를 클릭하여머리글 테두리 색상을 선택합니다 .
  5. 저장을 클릭합니다 .

커스텀 스타일링

사용자 지정스타일 옵션을 사용하면 액세스 제어 페이지에 대한 고급 사용자 지정을 수행할 수 있습니다. 예를 아래에서 볼 수 있습니다.

사용자 지정옵션을 사용하려면 다음 단계를 수행하십시오.

  1. 사용자 지정을 선택합니다 .
  2. 머리글 및 바닥 글에 사용자 정의 HTML을 입력하십시오.
  3. 저장을 클릭합니다.
